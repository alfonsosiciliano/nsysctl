NAME
     nsysctl - get or set kernel state

SYNOPSIS
     nsysctl [--libxo opts [-r]] [-deFIilmNqTtWy] [-V|v[h[b|o|x]]] [-IiqTW]
             [-B bufsize] [-f filename] name[=value] ...
     nsysctl [--libxo opts [-r]] [-deFIlmNqSTtWy] [-V|v[h[b|o|x]]] [-IqSTW]
             [-B bufsize] -A|a|X

DESCRIPTION
     The systems maintains a "Management Information Base" ("MIB") of kernel
     states.  The nsysctl tool can get the value and allows users with
     appropriate privilege to set kernel state.

     nsysctl shows only the information explicitly indicated by the options
     (see COMPATIBILITY for a comparison with sysctl(8).

     The following options are available:

     --libxo options
             Generate output via libxo(3) in a selection of different human
             and machine readable formats.  See xo_parse_args(3)

     -A      Doc.

     -a      Doc.

     -b      Doc

     -B bufsize
             [NOT IMPLEMENTED] DOC.

     -d      Doc.

     -e      Doc.

     -F      [DOC]

     -f filename
             [NOT IMPLEMENTED] Doc.

     -h      Doc.

     -I      [DOC]

     -i      Doc.

     -l      [DOC]

     -M      [DOC]

     -m      [DOC]

     -N      Doc.

     -o      Doc.

     -q      Doc.

     -S      [DOC]

     -T      Doc.

     -t      Doc

     -v      [DOC]

     -W      Doc.

     -X      Doc.

     -y      [DOC]

EXIT STATUS
     The nsysctl utility exits 0 on success, and >0 if an error occurs.

EXAMPLES
     To get the hostname:

           nsysctl -Nv kern.hostname

     To set the hostname:

           nsysctl kern.hostname=myBSD

     Dump all info about a state:

           nsysctl -NlydtmFv kern.ostype

     Dump all info about a state in xml format:

           nsysctl --libxo=xml,pretty -r -NlydtmFv kern.ostype

COMPATIBILITY
     The -w option has been deprecated and is silently ignored.

     % sysctl name

           % nsysctl -Nv name

     % sysctl -a:

           % nsysctl -NVa

     % sysctl -na:

           % nsysctl
